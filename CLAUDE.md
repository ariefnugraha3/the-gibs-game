## Key mechanics and conventions

- **Collisions are distance checks, not physics.** Zombie-hit, pickup, and blast use `distanceTo` radii. **Bullet-vs-zombie is a swept-segment check** (`segPointDist2`, prevâ†’current position; spawn-frame segment starts at the player's EYE so point-blank never tunnels). Survival: Monas AABB revert, solid trees/fountain cylinders (`resolveObstacles`), fence clamp. Campaign: walkable-union tests (`stage1Walk` grid / `highwayWalk`) with per-axis `slideWalk` (hug walls, don't stick) + rotated-AABB `resolveBlockers` (medians/wrecks/furniture; standable tops are walkable). Zombie bodies are solid to the player only (push-out at `CFG.zombie.bodyBlockRadius` 7.5, deliberately < claw-stop 8.0 so an attacking zombie doesn't shove the player every frame).
- **Frame-rate independence:** `dt` (clamped 0.05) and `step = dt*60`. Multiply motion by `step`; decrement timers by `dt`. Fire-rate/reload use real `Date.now()` time.
- **Entity model:** five arrays in `core/state.js` â€” `bullets, zombies, grenades, explosions, drops` â€” iterated backwards for splicing. `explosions` doubles as a generic visual-decay pool (ground puffs ride it). Blood is a **fixed pool of 14 sprites** (`effects.js`), explosion flashes a **fixed pool of 3 lights** â€” keep point-light counts constant (no mid-game shader recompiles). - **Zombies:** procedural blocky humans (`buildHumanZombie`, 6 profession variants, shared `ZG` geometries + per-instance materials â€” `disposeZombie` handles the head's material array; never dispose `ZG`/face texture). ## Editing guidance

- Keep the project a **static, buildless site**: no bundlers, frameworks, or npm dependencies (package.json stays metadata-only). It must run from a plain HTTP server.
- Follow the architecture rules at the top of [MODULES.md](MODULES.md) (scene hooks instead of mode if-else; CFG for tunables; live-binding state ownership; updateGame block order) and update MODULES.md whenever modules/exports/config keys change.
- Preserve existing controls (WASD move, Shift sprint, mouse look, left-click fire, right-click toggle ADS, 1/2/Q switch weapon, F melee, G grenade, R reload, C toggle crouch / Left-Ctrl hold crouch, Space jump / restart) and core mechanics unless the user asks for gameplay changes.
- If you edit the stage-1 room layout, re-verify grid connectivity (BFS from `S1_START` over floor cells) and that no furniture blocker parks in front of a doorway.

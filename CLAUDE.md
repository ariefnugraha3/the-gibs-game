# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- **Serve:** `python -m http.server 8000` (then open http://localhost:8000) — an HTTP server is MANDATORY; `file://` breaks ES modules + the config `fetch`. Internet required for the Three.js r128 CDN scripts.
- **Test:** `node tools/smoke.mjs` — headless smoke suite (zero deps, stubbed THREE/DOM driving the real `src/` modules). Run the WHOLE file; there is no per-test filter (comment out sections to isolate one).
- **Lint:** `node --check src/<file>.js` — per touched file (`"type": "module"` makes this work).
- **Mandatory per gameplay change:** add/adjust smoke asserts (keep them CONFIG-DRIVEN — read `CFG`, never hardcode tuned numbers) → `node tools/smoke.mjs` until green → `node --check` every touched file → sync CLAUDE.md + MODULES.md.

## Key mechanics and conventions

- **Collisions are distance checks, not physics.** Robot-hit, pickup, and blast use `distanceTo` radii. **Bullet-vs-robot is a swept-segment check** (`segPointDist2`, prev→current position; spawn-frame segment starts at the player's EYE so point-blank never tunnels). Survival: Monas is a solid AABB resolved PER-AXIS (`resolveMonas` in world.js — `MONAS_HALF` 22, base 44×44; hug-and-slide, NOT a full revert, so the player never sticks — mirrors the campaign wall fix), solid trees/fountain cylinders (`resolveObstacles`), fence clamp. **`bulletBlocked` follows the real Monas silhouette height-by-height** (base 44 low → thin obelisk ~16 at eye height, nothing above y 64) so bullets pass through the empty air beside the obelisk and hit robots behind — do NOT revert it to a flat 44-wide column. Campaign: walkable-union tests (`stage1Walk` grid / `highwayWalk`) with per-axis `slideWalk` (hug walls, don't stick) + rotated-AABB `resolveBlockers` (medians/wrecks/furniture; standable tops are walkable).
